# Hybrid Audio-Based Violence Detection System Design Document

**Date:** 2024-03-XX
**Topic:** Real-World Robustness via CMAG-v2 and Temporal Escalation
**Author:** Antigravity Agent (ML Engineer Role)
**Status:** Approved by User

## 1. Context and Problem Statement
The previous iteration of the CMAG (Cross-Modal Attention Gating) violence detection model relied heavily on "ideal conditions" (e.g., clean, isolated, 4-second audio clips with obvious punch or gunshot sounds) and oversimplified features (averaging 20 MFCCs across time and using a single scalar NLP score). Reviewers correctly pointed out that this approach fails in noisy, continuous, real-world environments where sounds overlap, speech in the background is confusing, and violence escalates over longer periods.

**Goal:** Engineer a production-ready hybrid violence detection system that fuses acoustic and linguistic features to reliably detect violence in messy, real-world streams.

## 2. Selected Architecture: Robust Late-Fusion Hybrid (CMAG-v2)

Based on the `ml-engineer` and `embedding-strategies` principles, we are implementing a robust late-fusion hybrid model consisting of four core components:

### A. The Audio Encoder (Acoustic Branch)
*   **Input:** 2D Mel-Spectrograms extracted from 4-second audio segments (16kHz).
*   **Model:** A pre-trained `ResNet-18` backbone (modified for 1-channel mel-spectrograms) mapping the spectrogram to a 128-dimensional dense vector (`audio_embed`). Note: This replaces the simpler custom CNN.
*   **Real-World Robustness (CRITICAL):** During training, we apply aggressive **Audio Data Augmentation** using the `audiomentations` library. This includes adding background noise (traffic, crowds), applying room reverberation, and randomly shifting pitch/time. This forces the CNN to learn invariant representations of violent impact and distress sounds rather than overfitting to pristine studio recordings.

### B. The Text/Linguistic Encoder (Linguistic Branch)
*   **Input:** Natural language transcriptions generated by the Whisper STT model.
*   **Model:** We depart entirely from the "single scalar NLP score." Instead, following proper `embedding-strategies`, we use a pre-trained transformer (`roberta-base`) to convert the text segment into a dense 768-dimensional semantic embedding (`text_embed`). Note: We swapped from `DeBERTa` and `DistilBERT` to `RoBERTa` to avoid Windows tokenization cache bugs.
*   **Real-World Robustness:** Dense embeddings capture nuanced contextual threat rather than relying on brittle keyword flags or simplistic 0/1 toxicity scores.

### C. Enhanced CMAG Fusion (CMAG-v2)
*   **Input:** The 128-dim `audio_embed` and the 768-dim (projected to 128) `text_embed`.
*   **Mechanism:** Bidirectional Cross-Modal Attention Gating. The audio embedding learns a gate (via a dense layer and Sigmoid) that modulates the text embedding, and vice versa. The gated features are concatenated (256-dim) and passed through a Fusion MLP.
*   **Output:** A 64-dimensional sequence-level feature vector representing the 4-second segment's multimodal state.

### D. Temporal Escalation Model
*   **Problem Solved:** Violence is rarely an isolated 4-second event; it's a sustained pattern. The old model used a post-hoc Weighted Moving Average (WMA) script.
*   **Model:** A Bidirectional LSTM (Long Short-Term Memory) network.
*   **Mechanism:** It consumes a sliding temporal window of the CMAG-v2 segment outputs (e.g., the last 5 segments = 20 seconds of context). The Bi-LSTM learns the *escalation trajectory* of the violence.
*   **Output:** The final violence probability logic.

## 3. Data Pipeline and Training Strategy
We will train on all available datasets (`audio_violence`, `cremad`, `esc50`, `urbansound8k`, `jigsaw_toxic`), mapped to a unified `Violent/Safe` taxonomy.

**Training Phases (ML Engineer Approach):**
1.  **Pre-training Encoders (Optional but recommended):** Train the ResNet-18 audio encoder independently on the Audio-only dataset split. Fine-tune the RoBERTa Text Embedding model on Jigsaw Toxic data.
2.  **End-to-End Fusion Training:** Freeze (or use low learning rate for) the encoders, and aggressively train the CMAG-v2 gating layers and MLP using augmented audio inputs.
3.  **Temporal Training:** Freeze the segment-level CMAG framework and train the Bi-LSTM on continuous "streams" synthesized by padding violent clips with safe background noise.

## 4. Evaluation and Verification
Replacing the old evaluation scripts with proper MLOps metrics:
*   Offline static clip evaluation (Confusion Matrix, ROC-AUC, PR-AUC).
*   **Streaming Evaluation (New):** Testing the full Temporal pipeline on long (1+ minute) continuous audio tracks containing multiple noisy background events simulating real-world scenarios.

## 5. Deployment — Full-Stack Web Application
To demonstrate the system in real-world conditions, we will build a full-stack web application:
*   **Backend (FastAPI):** Serves the full PyTorch inference pipeline. Provides stateless REST endpoints for static files and a stateful WebSocket endpoint for real-time streaming.
*   **Frontend (Standard React.js):** A clean, simple UI featuring two core components:
    1.  **File Upload Tile:** For batch processing of pre-recorded audio files.
    2.  **Live Microphone Tile:** Captures mic audio via the browser, streams it to the WebSocket, and dynamically displays the per-segment inference results (Timestamp, Audio Score, NLP Score, Final state).
